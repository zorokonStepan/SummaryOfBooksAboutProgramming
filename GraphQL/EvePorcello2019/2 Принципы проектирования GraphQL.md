```
    1. Иерархичность. 
       GraphQL-запрос является иерархическим.
       Поля встраиваются в другие поля, и запрос формируется подобно данным, которые он возвращает
    2. Ориентированность на продукт. 
       GraphQL управляется потребностями данных клиента, а также языком и временем выполнения, поддерживаемыми клиентом.
    3. Строгая типизация. 
       Сервер GraphQL поддерживается системой GraphQL. 
       В схеме каждая точка данных имеет определенный тип, насчет которого она будет проверена.
    4. Запросы, определенные клиентом. 
       Сервер GraphQL предоставляет возможности, которые клиенты могут использовать.
    5. Интроспектива. 
       Язык GraphQL может запрашивать серверную систему типов GraphQL.
```
