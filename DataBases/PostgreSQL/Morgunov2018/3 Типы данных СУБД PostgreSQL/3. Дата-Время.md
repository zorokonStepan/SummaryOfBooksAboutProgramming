```
    1. date                 # default:«yyyy-mm-dd»
    2. time                 # хранит время суток
    3. time with time zone  # хранит время суток + часовой пояс
    4. timestamp            # временная отметка = дата + время
    5. timestamp with time zone(timestamptz)
    6. interval  # продолжительность отрезка времени между двумя моментами времени
    
    документация на PostgreSQL не рекомендует использовать тип time with time zone
```

```
    Важно помнить, что при вводе значений их нужно заключать в одинарные кавычки, как и текстовые строки.
    
     По умолчанию используется формат: «yyyy-mm-dd».
     SELECT '2016-09-12'::date;
     
     ::date - операция приведения типа
```

### Функции для работы с датой и временем
```
    1. current_date - получение значения текущей даты (при ее вызове круглые скобки не используются)
        SELECT current_date;

    2. to_char - Если нам требуется вывести дату в другом формате
        SELECT to_char( current_date, 'dd-mm-yyyy' );
        
    3. current_time - получение значения текущего времени (при ее вызове круглые скобки не используются)
        SELECT current_time;
        
    4. current_timestamp - Для получения значения текущей временной отметки (т.е. даты и времени в одном значении)
       (при ее вызове круглые скобки не используются)
       SELECT current_timestamp;

    5. date_trunc - Значения временных отметок можно усекать с той или иной точностью
        SELECT ( date_trunc( 'hour', current_timestamp ) );
        
    6. extract - Из значений временных отметок можно извлекать отдельные поля, т.е. год, месяц, день, число часов, 
       минут или секунд и т.д.
       SELECT extract( 'mon' FROM timestamp '1999-11-27 12:34:56.123459' ); 
    
```
